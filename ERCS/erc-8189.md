## Abstract

[ERC-8188](./eip-8188.md) defines a general Physical Commodity-Backed Token (PCBT) standard. This ERC extends it with gold-specific conventions required to build interoperable, regulated, and Shariah-compliant tokenized gold products targeting the $35 trillion physical gold market.

This ERC standardizes:

- **Canonical micro-unit system** — micrograms (µg) as the internal weight unit for lossless integer arithmetic
- **Unit conversions** — troy ounce ↔ gram ↔ tola ↔ Islamic gold dinar (4.25g)
- **LBMA Good Delivery compliance** — on-chain validation of London Bullion Market Association bar requirements
- **Shariah compliance interface** — fully allocated, no riba (interest), no maysir (speculation)
- **Islamic Gold Dinar** — 4.25g (one mithqal) as a standardized redemption denomination
- **Gold price oracle integration** — Chainlink XAU/USD feed with staleness protection
- **Purity differentiation** — 999.9 (LBMA), 916 (22K dinar), 750 (18K) in machine-readable form

Implementations of [ERC-8189](./eip-8189.md) MUST also implement ERC-8188.

## Motivation

Gold has specific characteristics that a general PCBT standard cannot fully address:

**1. Unit fragmentation.** Gold is traded in troy ounces globally (LBMA standard), grams locally (Malaysia, Indonesia, Gulf), and tolas in South Asia. Wallets and DeFi protocols need native conversion between all units. No existing token standard defines this.

**2. Shariah compliance.** 1.8 billion Muslims require gold investments to be fully allocated (no fractional reserve), free from riba (interest), and free from maysir (speculation). No existing ERC defines a Shariah compliance interface. Malaysia, UAE, Saudi Arabia, and Indonesia represent over $3 trillion in Islamic finance assets that require this specification.

**3. Islamic Gold Dinar.** The 4.25g gold dinar is a legally recognized unit in Malaysia (Bank Negara Malaysia, Kelantan Dinar Act) and is traded across the Gulf. Standardizing the dinar as a redemption denomination unlocks Islamic fintech use cases that no existing token supports.

**4. LBMA Good Delivery on-chain.** The LBMA Good Delivery standard (350–430 troy oz, ≥99.5% purity) is the global institutional benchmark. Encoding its requirements on-chain enables trustless verification by DeFi protocols and institutional custodians.

**5. XAU/USD oracle standard.** Every gold DeFi application independently sources the gold price. Standardizing the oracle interface and staleness check eliminates a repeated source of protocol-specific risk.

## Specification

The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHOULD", "MAY" in this document are to be interpreted as described in RFC 2119 and RFC 8174.

### 1. Canonical Weight Unit: Microgram (µg)

All weights in ERC-8189 MUST be expressed internally in **micrograms** for lossless arithmetic.

```
1 troy ounce    = 31,103,477 µg  (31.1034768g, rounded)
1 gram          = 1,000,000 µg
1 tola          = 11,663,800 µg  (11.6638g)
1 Islamic Dinar = 4,250,000 µg   (4.25g, one mithqal)
1 mithqal       = 4,250,000 µg   (equal to dinar)
```

### 2. `IGoldToken` Interface

```solidity
// SPDX-License-Identifier: CC0-1.0
pragma solidity ^0.8.20;

import "./IERC8188.sol";

interface IGoldToken is ICommodityToken {
    // ── Unit Constants ────────────────────────────────────────────────────────
    /// @notice Micrograms per troy ounce (31,103,476.8 → stored as 31,103,477)
    function MICRO_PER_TROY_OZ() external pure returns (uint256);
    /// @notice Micrograms per gram (1,000,000)
    function MICRO_PER_GRAM()    external pure returns (uint256);
    /// @notice Micrograms per tola (11,663,800)
    function MICRO_PER_TOLA()    external pure returns (uint256);
    /// @notice Micrograms per Islamic gold dinar (4,250,000 = 4.25g)
    function MICRO_PER_DINAR()   external pure returns (uint256);
    /// @notice Base token units for exactly 1 gram (implementation-specific)
    function TOKENS_PER_GRAM()   external pure returns (uint256);

    // ── Unit Conversions ──────────────────────────────────────────────────────
    function toMicrograms(uint256 baseUnits)         external view returns (uint256);
    function fromMicrograms(uint256 micrograms)      external view returns (uint256);
    /// @notice Returns grams scaled × 1e6 (e.g. 31103477 for 31.103477g)
    function toGramsScaled(uint256 baseUnits)        external view returns (uint256);
    /// @notice Returns troy ounces scaled × 1e8
    function toTroyOuncesScaled(uint256 baseUnits)   external view returns (uint256);
    /// @notice Returns tolas scaled × 1e8
    function toTolasScaled(uint256 baseUnits)        external view returns (uint256);
    /// @notice Returns Islamic dinars scaled × 1e4
    function toDinarsScaled(uint256 baseUnits)       external view returns (uint256);

    // ── Purity ────────────────────────────────────────────────────────────────
    /// @notice Default lot purity in basis points (e.g. 9999 = 99.99%)
    function purityBps() external view returns (uint256);
    /// @notice Pure gold micrograms from alloy weight and purity
    function pureGoldMicro(uint256 alloyMicro, uint256 purityBps_)
        external pure returns (uint256);

    // ── LBMA Compliance ───────────────────────────────────────────────────────
    /// @notice Returns true if the lot meets LBMA Good Delivery requirements
    /// @dev Weight 350–430 troy oz, purity ≥ 99.5%, branded refinery
    function isLBMACompliant(uint256 lotId) external view returns (bool);
    /// @notice LBMA accreditation status of the primary custodian vault
    function lbmaAccredited() external view returns (bool);
    /// @notice Primary custodian vault name
    function custodianName()  external view returns (string memory);

    // ── Gold Price Oracle ─────────────────────────────────────────────────────
    /// @notice Current XAU/USD price from oracle (8 decimals, Chainlink format)
    /// @dev MUST revert or return 0 if updatedAt > block.timestamp - maxStaleness
    function goldPriceUSD()
        external view returns (uint256 pricePerOz, uint256 updatedAt);
    /// @notice Gold price per gram in USD (8 decimals)
    function goldPricePerGramUSD() external view returns (uint256);
    /// @notice Token value in USD for a given token amount (8 decimals)
    function tokenValueUSD(uint256 tokenAmount) external view returns (uint256);
    /// @notice Chainlink AggregatorV3Interface address for XAU/USD
    function priceOracle() external view returns (address);
    /// @notice Maximum allowed oracle staleness in seconds (SHOULD be ≤ 3600)
    function maxOracleStaleness() external view returns (uint256);

    // ── Shariah Compliance ────────────────────────────────────────────────────
    /// @notice Returns true if the implementation is Shariah-compliant
    /// @dev Requires: fully allocated, no riba, no maysir, no gharar
    function shariahCompliant() external view returns (bool);
    /// @notice Identifying string of Shariah advisory body
    function shariahAdvisory()  external view returns (string memory);
    /// @notice Returns true if totalSupply × microPerToken ≤ totalVerifiedReserve
    function isFullyAllocated() external view returns (bool);

    // ── Islamic Gold Dinar ────────────────────────────────────────────────────
    /// @notice Token amount for N gold dinars (4.25g each)
    function dinarToTokens(uint256 dinarCount)   external view returns (uint256);
    /// @notice Full gold dinars representable by tokenAmount (floor)
    function tokensToDinars(uint256 tokenAmount) external view returns (uint256);
    /// @notice Minimum tokens required to redeem one gold dinar
    function dinarRedemptionMinimum()            external view returns (uint256);

    // ── Events ───────────────────────────────────────────────────────────────
    event GoldPriceUpdated(uint256 pricePerOz, uint256 pricePerGram, uint256 updatedAt);
    event ShariahStatusUpdated(bool compliant, string advisory);
    event DinarRedemptionRequested(
        uint256 indexed requestId,
        address indexed requester,
        uint256 dinarCount,
        uint256 tokensBurned
    );
}
```

### 5. `GoldUnits` Library

```solidity
library GoldUnits {
    uint256 internal constant MICRO_PER_TROY_OZ = 31_103_477;
    uint256 internal constant MICRO_PER_GRAM    = 1_000_000;
    uint256 internal constant MICRO_PER_TOLA    = 11_663_800;
    uint256 internal constant MICRO_PER_DINAR   = 4_250_000;

    function microToGramsScaled(uint256 micro)
        internal pure returns (uint256) { return micro; }

    function microToTroyOzScaled(uint256 micro)
        internal pure returns (uint256) { return (micro * 1e8) / MICRO_PER_TROY_OZ; }

    function microToDinarsScaled(uint256 micro)
        internal pure returns (uint256) { return (micro * 1e4) / MICRO_PER_DINAR; }

    function microToTolasScaled(uint256 micro)
        internal pure returns (uint256) { return (micro * 1e8) / MICRO_PER_TOLA; }

    function gramsToMicro(uint256 grams)
        internal pure returns (uint256) { return grams * MICRO_PER_GRAM; }

    function dinarsToMicro(uint256 dinars)
        internal pure returns (uint256) { return dinars * MICRO_PER_DINAR; }

    function pureGoldMicro(uint256 alloyMicro, uint256 purityBps)
        internal pure returns (uint256) { return (alloyMicro * purityBps) / 10_000; }

    /// @notice Validate LBMA Good Delivery requirements
    /// @dev Weight: 350–430 troy oz; Purity: ≥99.5%
    function isLBMAGoodDelivery(uint256 weightTroyOzScaled, uint256 purityBps)
        internal pure returns (bool) {
        return weightTroyOzScaled >= 35_000_000_000   // 350 × 1e8
            && weightTroyOzScaled <= 43_000_000_000   // 430 × 1e8
            && purityBps >= 9950;
    }
}
```

### Shariah Compliance Requirements

Implementations returning `shariahCompliant() == true` MUST:

1. **Fully allocated**: Every token maps to a specific physical gold lot. `isFullyAllocated()` MUST return `true` at all times. Fractional-reserve minting is forbidden.
2. **No riba**: The contract MUST NOT accrue, pay, or require interest. Storage fees, if any, MUST be flat-rate service fees, not interest-bearing.
3. **No maysir**: The contract MUST NOT include derivatives, leverage, or speculation instruments.
4. **No gharar**: Physical gold backing MUST be verifiable via `IProofOfReserve` at all times.
5. **Advisory**: `shariahAdvisory()` MUST return a non-empty string identifying the certifying body.

### Gold Price Oracle

Implementations MUST:
- Source XAU/USD from a `AggregatorV3Interface`-compatible oracle (Chainlink format)
- Return 8-decimal prices consistent with Chainlink conventions
- Check `updatedAt > block.timestamp - maxOracleStaleness` and revert or return 0 if stale
- Expose `maxOracleStaleness()` — SHOULD be ≤ 3600 seconds

## Rationale

**Why micrograms as canonical unit?** Micrograms are the finest precision needed for gold (1 µg = 0.000001g). Using micrograms as the canonical unit allows exact integer arithmetic for all standard denominations: 1g = 1,000,000, 1 dinar = 4,250,000, 1 troy oz = 31,103,477. No floating point is required anywhere in the stack.

**Why Islamic Gold Dinar as a standardized denomination?** The 4.25g dinar (one mithqal) is a legally recognized monetary unit in Malaysia, Kelantan, and across the Gulf Cooperation Council. It represents a natural minimum redemption unit for Shariah-compliant gold products. Without standardization, each issuer defines its own redemption units, creating fragmentation.

**Why `shariahCompliant()` as a boolean view function?** This enables DeFi protocols to filter for compliant products programmatically. Wallets in Islamic-majority markets (Indonesia, Malaysia, Saudi Arabia, UAE) can surface only compliant products. The Shariah compliance status is attested on-chain, with the advisory body identified in `shariahAdvisory()`.

**Why separate LBMA validation logic?** LBMA Good Delivery is the de facto global standard for institutional gold. Encoding its requirements (350–430 oz weight range, ≥99.5% purity) as verifiable on-chain conditions allows custodians, prime brokers, and DeFi protocols to trustlessly verify bar quality without relying on off-chain attestations.

## Backwards Compatibility

ERC-8189 is an additive extension of ERC-8188 and [ERC-20](./eip-20.md). No breaking changes. All existing wallets, DEXes, and DeFi protocols interact with ERC-8189 tokens as standard ERC-20 tokens. Extensions are discoverable via ERC-165.

## Security Considerations

**Oracle manipulation.** `goldPriceUSD()` relies on an external Chainlink feed. Implementations MUST validate staleness. The `maxOracleStaleness` SHOULD be set ≤ 3600 seconds. Price spikes or oracle failures MUST NOT allow unbacked minting.

**Dinar weight precision.** 4.25 grams = 4,250,000 micrograms. Integer division is exact. Implementations MUST NOT use floating point for unit conversions.

**Reserve ratio invariant.** Implementations MUST check `totalSupply() × microPerToken ≤ IProofOfReserve.totalVerifiedReserve()` within every `mintFromLot` call and MUST revert if this invariant would be violated.

**Purity calculation.** All purity calculations MUST use integer basis points. `(alloyMicro × purityBps) / 10_000` is the canonical formula. Never use floating point.
