## Abstract

This ERC defines a minimal extension to ERC-20 that exposes a canonical disclosure URI. The URI provides a standard mechanism for discovering issuer-provided disclosures associated with tokenised real-world assets, such as compliance information, supporting documentation, and other relevant off-chain disclosures.

## Motivation

Tokenised real-world assets require a consistent and discoverable mechanism for wallets, explorers, indexers, and other applications to locate issuer disclosures.

Today, disclosure information is typically published through issuer websites or documentation with no standard on-chain discovery mechanism. As a result, applications must rely on issuer-specific conventions.

This ERC standardises a minimal interface for discovering issuer disclosures while intentionally avoiding any requirements on disclosure format, regulatory framework, or validation of the referenced content.

## Specification

The key words **"MUST"**, **"MUST NOT"**, **"REQUIRED"**, **"SHALL"**, **"SHALL NOT"**, **"SHOULD"**, **"SHOULD NOT"**, **"RECOMMENDED"**, **"NOT RECOMMENDED"**, **"MAY"**, and **"OPTIONAL"** in this document are to be interpreted as described in RFC 2119 and RFC 8174 when, and only when, they appear in all capitals.

ERC-20 contracts implementing this standard MUST implement the following interface.

The returned URI MAY use any URI scheme.

An empty string indicates that no disclosure URI is available.

This standard does not prescribe the format or contents of the referenced disclosure resource.

```solidity
// SPDX-License-Identifier: CC0-1.0
pragma solidity ^0.8.20;

interface IERC20RwaDisclosureURI {
    /// @notice Returns the canonical disclosure URI.
    function disclosureURI() external view returns (string memory);
}
```

## Rationale

This standard intentionally defines only a read interface.

It does not specify how a disclosure URI is stored or updated, allowing implementations to use immutable values, upgradeable contracts, governance mechanisms, or other implementation-specific approaches.

Similarly, this ERC does not prescribe the format of the referenced disclosure resource. Existing regulatory frameworks, industry standards, and issuer requirements may evolve independently of this interface without requiring changes to the ERC itself.

## Backwards Compatibility

This ERC is fully backwards compatible with ERC-20.

It introduces a single optional view function and does not modify token behaviour, transfer semantics, or existing ERC-20 interfaces.

## Security Considerations

The disclosure URI references off-chain information whose availability, integrity, and authenticity depend on external systems.

Applications consuming the URI should independently evaluate the trustworthiness of the referenced resource and the identity of the entity publishing it.

Because the referenced content may change over time, consumers should not assume that previously retrieved disclosures remain current.

This ERC does not provide guarantees regarding the accuracy, completeness, legal validity, or permanence of any referenced disclosure.
